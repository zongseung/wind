## 2. 허용 vs 금지 (핵심 표)

| 항목 | 판정 | 근거/설명 |
|---|---|---|

| **학습 기간 SCADA**로 라벨 정제(정지구간 마스킹) | ✅ 허용 | 학습 데이터 품질 개선 |
| info.xlsx(좌표·허브높이·용량) 사용 | ✅ 허용 | 메타 정보로 제공됨 |
| NWP끼리 파생(전단·밀도·불일치·공간 gradient 등) | ✅ 허용 | NWP 입력만으로 계산 |
| 학습 SCADA로 NWP 풍속 bias 학습→테스트 적용(MOS) | ⚠️ 문구상 허용, **팀 결정상 미사용** | 3절 참조 |
| **평가 기간(2025) SCADA·실측·운영데이터** | ❌ 금지 | 애초에 미제공 + §12 |
| **외부 재분석자료(ERA5·MERRA 등)** 를 입력으로 | ❌ 금지 | §12 "재분석자료" |
| **사후 보정된 NWP**(관측으로 후처리된 예보) 입력 | ❌ 금지 | §12 "사후 보정자료" |
| 테스트 라벨을 본 어떤 튜닝/선택 | ❌ 금지 | 표준 누설 |

## 3. "보정(MOS)"에 대한 팀 결정 — 미사용
- **문구상**: 학습 SCADA로 NWP 오차를 학습해 테스트 NWP에 적용하는 MOS는, 입력이 "테스트 NWP + 학습 파라미터"뿐이라 규칙 위반이 아니다.
- **그럼에도 쓰지 않는 이유 2가지**:
  1. **해석 리스크** — "보정"이 §12 "사후 보정자료"와 어감상 가까워 시비 소지. 애매하면 안전하게.
  2. **중복** — GBM을 `(테스트 가용 NWP) → (실제 발전량)`으로 학습하면 NWP 계통오차를 **모델이 이미 end-to-end 흡수**. 별도 MOS 이득이 작다.
- **결론**: 독립적 NWP "보정" 단계는 두지 않는다. SCADA는 아래 4절 용도로만.

---

## 4. SCADA 사용 정책 (학습 데이터만)
- ✅ **경험적 파워커브 prior**: SCADA 풍속→발전량 곡선 학습 → 테스트 NWP 허브풍속(117m 외삽)에 적용해 feature로.
- ✅ **라벨 정제**: "바람 충분(≥cut-in)한데 발전 0"인 정지/제한 구간을 마스킹·가중 축소.
- ❌ **테스트 입력으로는 절대 사용 금지** (테스트엔 SCADA 없음).
- 참고: "바람 있는데 발전 0"(고장/제한)은 2025년에 예측 불가한 **원리적 오차 하한**(EDA상 ~6~9%). 국면(regime) 정의에 이 신호를 넣지 말 것 → HMM regime은 **NWP 변수로만** 정의.

## 5. 예보 타이밍 (누설 방지 시간 정렬)
- NWP는 매일 **09:00 KST 초기화**, 해당일 **13:00 KST부터 사용 가능**(`data_available_kst_dtm`).
- 다음날 01:00 ~ 그 다음날 00:00의 24시간이 같은 `data_available` 값을 공유.
- 즉 리드타임 약 12~36시간. **feature는 `data_available` 시점 이후 정보만** 써야 하며, 미래 예보 시각의 라벨/실측을 당겨 쓰면 안 됨.

## 7. 평가지표 정합 (모델·후처리 규율)
- **NMAE = MAE / 설비용량**. 그룹 내에선 MAE 최소화 = NMAE 최소화 → 손실 **MAE(L1)**.
- 설비용량 kWh 기준: 그룹1/2 = 21,600, 그룹3 = 21,000.
- **유효구간**: 실제 발전량 **≥ 설비용량의 10%** 구간만 채점 → 저출력 구간 가중 축소 고려.
- **FICR**: 6%/8% 오차경계 정확도 인센티브 → 마지막 후처리에서만 제한 적용(전체 적용은 NMAE 악화 가능).
- **필수 후처리**: 예측을 **[0, 설비용량 kWh]로 클리핑**. isotonic 파워커브 캘리브레이션 권장.

## 8. 검증 규율
- **랜덤 분할 금지** (강한 계절성 → 점수 부풀림). 시간 순서 보존.
- 권장: **2024 holdout** + 계절 블록 CV. 그룹3은 2023학습/2024검증.
- 과거 fold 정보만으로 bias 보정(미래 누설 차단).
